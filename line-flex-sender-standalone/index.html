⚠️ 事前準備：CORS制約の回避（検証用ブラウザの起動）

このツールはブラウザから直接 g123 の Broadcast API にリクエストを送ります。通常のChromeではCORSエラーが発生するため、以下の手順で検証用Chromeを起動してから使用してください。

  1. キーボードの Win + R を押して「ファイル名を指定して実行」を開く。
  2. 以下のコマンドをコピーして貼り付け、「OK」を押す。
    chrome.exe --user-data-dir="C:\ChromeDev" --disable-web-security --disable-site-isolation-trials
  3. 新しく開いたChromeでこのツールのURLを開きなおす。

※このChromeでは通常のネットサーフィンをしないでください。

⚠️ 事前準備：CORS制約の回避（検証用ブラウザの起動）

ブラウザから直接LINEのAPIを叩くため、通常のChromeでは通信がブロック(CORSエラー)されます。以下の手順でセキュリティ設定を解除した検証用Chromeを起動してからこのツールを使用してください。

  1. キーボードの Win + R を押して「ファイル名を指定して実行」を開く。
  2. 以下のコマンドをコピーして貼り付け、「OK」を押す。
    chrome.exe --user-data-dir="C:\ChromeDev" --disable-web-security --disable-site-isolation-trials
  3. 新しく開いたChromeでこのツールのURLを開きなおす。

※LINE以外への通常のネットサーフィンにはこのChromeを使用しないでください。

※ユーザーIDはLINE Developersコンソールの「チャネル基本設定」一番下で確認できます。
※上限を指定した場合、LINEの仕様により「属性が推定可能なユーザー」の中からランダムに指定人数まで配信されます（Narrowcast APIを使用）。

📖 取扱説明書

1
検証用 Chrome を起動する

ブラウザからLINE APIに直接リクエストを送るにはCORS制限を解除する必要があります。Win + R を押し、以下のコマンドを実行してください。

chrome.exe --user-data-dir="C:\ChromeDev" --disable-web-security --disable-site-isolation-trials
※この検証用Chromeでのみ本ツールを使用してください。通常のネットサーフィンには使用しないでください。
2
チャネルアクセストークンを入力する

LINE Developers コンソール → 対象チャネル → 「Messaging API設定」タブ → 「チャネルアクセストークン（長期）」を発行してコピーします。
「Bearer 」は不要です。トークン本文のみ入力してください。

3
（テスト送信の場合）ターゲットユーザーIDを入力する

LINE Developers コンソール → 対象チャネル → 「チャネル基本設定」タブ → 最下部の「あなたのユーザーID」を確認してください。U から始まる文字列です。

テスト送信は 本番配信前に自分のアカウントで必ず確認 することをおすすめします。
4
Flex Message JSON を貼り付ける

LINE Flex Message Simulator でメッセージを作成し、「JSON」タブに表示されるJSONをコピーして左のテキストエリアに貼り付けます。右側にリアルタイムプレビューが表示されます。

対応フォーマット：
・{"type":"flex","altText":"...","contents":{...}}（メッセージオブジェクト全体）
・{"type":"bubble",...} または {"type":"carousel",...}（contentsの中身のみ）
5
altText を設定する

プッシュ通知やトークリストに表示される代替テキストです。JSONに altText が含まれている場合は自動入力されます。任意で書き換えてください。

6
送信する

目的に応じていずれかのボタンを押してください。

特定ユーザーにテスト送信（Push API）：手順3で入力したユーザーIDにのみ送信。確認作業に使用します。

本番配信（Broadcast / Narrowcast）：
 ・配信数上限が空欄の場合 → 全友だちに一斉配信（Broadcast API）
 ・配信数上限を入力した場合 → 属性推定可能なユーザーの中から最大N人にランダム配信（Narrowcast API）
⚠️ 本番配信ボタンは確認ダイアログが表示されますが、押したら即送信されます。本番配信前に必ずテスト送信で見た目を確認してください。
7
レスポンスを確認する

送信後、ページ下部に LINE API からのレスポンスが表示されます。HTTP Status: 200 であれば送信成功です。エラーの場合はメッセージを確認してください。

よくあるエラー：
・CORSエラー：検証用Chromeを使用していない → 手順1を確認
・401 Unauthorized：アクセストークンが間違っている
・400 Bad Request：JSONの形式が不正（プレビューでエラーが出ていないか確認）
